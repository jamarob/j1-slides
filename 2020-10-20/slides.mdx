# prop-types

-   `$ npm install prop-types`
-   Typechecking for props
-   But only **at runtime**!
-   Mark props as optional or required
-   Fail faster, find bugs more quickly
-   Type information can be used by other software (e.g. storybook)

---

## Writing simple prop-types

Components may have a `propTypes` property with key-value pairs describing its props and their type.

```jsx
Greeting.propTypes = {
        name: PropTypes.string,
};

export default function Greeting({ name = "Stranger" }) {
        return <h1>Hello {name}!</h1>;
}
```

Check out the usage section at [npm](https://www.npmjs.com/package/prop-types#usage) for a complete overview of available types.

---

## Writing complex prop-types

We are not limited to primitives for prop-types.

```jsx
Comments.propTypes = {
	      posts: PropTypes.arrayOf(
		            PropTypes.shape({
                        author: PropTypes.string.isRequired,
			                  likes: PropTypes.number,
                        published: PropTypes.instanceOf(Date)
		            })
	      )
}
```

# CSS-Grid
Two dimensional layouts oriented in columns and rows. 

The **grid container** is the element with `display: grid`, all of its direct children are called **grid items**. 

---

## Grid container properties I

|Property|Value|
|---|---|
|display|grid|
|grid-gap|15px 10px |
|grid-template-rows | 48px 30% 1fr min-content|
|grid-template-columns | repeat(5, 1fr) |

---

## Grid container properties II
|Property|Value|
|---|---|
|justify-items| start \| end \| center \| stretch (default)|
|align-items|  start \| end \| center \| stretch (default)|
|grid-auto-columns | implicit size of grid tracks|
|grid-auto-rows | implicit size of grid tracks |

---

## Grid item properties

|Property|Value|
|---|---|
|grid-column| start-line / end-line/span |
|grid-row| start-line / span|
|justify-self | start \| end \| center \| stretch (default)|
|align-self | start \| end \| center \| stretch (default) |
